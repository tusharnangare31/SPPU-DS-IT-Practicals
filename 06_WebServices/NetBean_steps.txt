# Part 4 — If HTTP 500 appears

## Check 1

Open:

```txt id="e1"
build\web\WEB-INF\wsdl
```

If empty → this is the same bug we found.

---

## Fix

Open `CalculatorServlet.java`.

Remove:

```java id="e2"
import javax.xml.ws.WebServiceRef;
```

Remove:

```java id="e3"
@WebServiceRef(...)
```

Replace:

```java id="e4"
private MyCalculatorWebService_Service service;
```

with:

```java id="e5"
private MyCalculatorWebService_Service service =
        new MyCalculatorWebService_Service();
```

This avoids the NetBeans/GlassFish WSDL-copy bug.

# Quick error mapping

| Error                                  | Cause                          | Fix                     |
| -------------------------------------- | ------------------------------ | ----------------------- |
| `package com.myservice does not exist` | Old reference deleted          | Drag service again      |
| `HTML lang attribute error`            | Used `?Tester` instead of WSDL | Use `?wsdl`             |
| `WEB-INF/wsdl file not found`          | NetBeans didn’t copy WSDL      | Remove `@WebServiceRef` |
| `MyCalculatorWebService_Tester`        | Imported tester page           | Delete and re-add       |
| `HTTP 500 @WebServiceRef`              | Runtime injection failure      | Manual `new Service()`  |

If you do these steps in order, the same fix works for **Addition, Subtraction, String operations, Odd/Even, Prime, Calculator**, etc.
